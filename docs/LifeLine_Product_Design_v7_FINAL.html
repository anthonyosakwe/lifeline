LifeLine.

Twelve screens. Three flows. One promise — your contacts know you're safe before you have to ask for help.

LifeLine is a personal safety and check-in app for Nigerian university students. This design pack documents every screen in the MVP — from first launch to emergency resolution — designed prevention-first, with SOS as the safety net.

Primary persona Ada Umeh — 20, 300-level Biochemistry, University of Nigeria, Nsukka. Walks back to her hostel after evening labs and study sessions; carries a Tecno on MTN 3G/4G.

User Journey

Five screens. Ninety seconds.

User taps the app icon, lands on the splash, sees the value prop, verifies their number, sees the second value prop, and adds at least one trusted contact. They cannot reach the home screen with zero contacts — this is the hard gate that converts "downloaded" into "actually useful."

Privacy & Failure Paths

Consent before contact. Recovery before frustration.

  • Wrong OTP — input boxes shake, error toast appears, attempts counter increments. Three failures → 5-minute lockout.
  • Contact consent — trusted contacts must accept via SMS link before they appear with a tick. Pending contacts cannot receive alerts.
  • Skip the value props — both onboarding slides have a quiet "Skip" so returning users aren't punished.

Why it works

Prevention before emergency.

Most safety apps optimise for the worst moment. LifeLine optimises for the thousand normal moments that come before it. A user who shares one trip a day forms a habit; a habit creates standing trust with two contacts; standing trust means an emergency alert is taken seriously the first time.

Failure Paths

When things go quietly wrong.

  • GPS denied or unavailable — bottom-sheet modal offers manual entry. The trip can still be shared with an address.
  • ETA hits 0:00 without arrival — red "OVERDUE" banner replaces the countdown; contacts are auto-alerted at minute +5.
  • User closes the app mid-trip — tracking continues via service worker. Resumed automatically on relaunch.

Failure mode by design

The three-second rule.

Phone in pocket. Bump. Tap. Slide. A typical false-positive flow lasts under a second. LifeLine's SOS requires three seconds of continuous pressure — long enough to be deliberate, short enough not to feel like an obstacle when you actually need it.

Cancellation as feature

2:45 to change your mind.

Contacts are alerted instantly — the cost of being wrong is "Thank God, false alarm." The cost of being too slow is real harm. The 2:45 grace window catches accidental activations without making the user fight the UI when seconds matter.

01
Dark-first
Navy background designed for night-safety use. High contrast guarantees visibility in dim or stressful conditions.
02
Thumb-reachable
Every primary action sits in the bottom third. SOS button is 48dp+ for one-handed operation under stress.
03
Prevention > Panic
Safe Arrival is the daily habit driver. SOS is the safety net. The UI hierarchy reflects this on every screen.
04
Network-resilient
Leaflet embedded inline — zero CDN dependency. Tiles fail gracefully to grey. Tested on MTN 3G in Abuja and Lagos.

Accessibility log.

Audit against WCAG 2.1 AA baseline. Pass = meets or exceeds. Partial = MVP-acceptable; queued for v2.

Colour contrast (1.4.3)
White on navy = 14.8:1 (AAA). Teal CTA on dark navy = 8.7:1 (AAA). Red on dark red SOS screen = 6.2:1 (AA Large).
✓ PASS
Touch target size (2.5.5)
SOS button 78dp. Primary CTAs ≥44dp height. ETA chips 36dp tap area with 5dp gap. Avatars on response cards are non-interactive.
✓ PASS
Text size (1.4.4)
Body text 13–15px. Hero/countdown text scales up to 52px. Respects browser zoom up to 200%. No fixed pixel layouts that break.
✓ PASS
Colour not sole indicator (1.4.1)
Status always paired with text + icon: "✓ Consent confirmed" / "⏳ Consent pending" / "● SOS LIVE". Safe/danger never communicated by colour alone.
✓ PASS
Keyboard navigation (2.1.1)
All interactive elements are native buttons/links. Tab order follows visual flow. Focus rings visible on every focusable element.
✓ PASS
Screen reader labels (1.3.1, 4.1.2)
aria-label on SOS button ("Hold three seconds to activate emergency"). All icons paired with visible text. Map alt text describes route.
✓ PASS
Reduced motion (2.3.3)
prefers-reduced-motion media query disables confetti, ring fill animation, and pulsing dots. Critical animations (countdown) remain.
✓ PASS
Haptic feedback (extra)
SOS activation triggers vibration pattern [200,100,200]. Helps low-vision users and confirms action when phone is in pocket.
✓ PASS
